CSS Layouts

Introduction

At this point, you should understand the Box Model, and how it can help space out your elements to create some simple layouts, like in our recipe page example. We saw how using basic CSS styles can create pretty presentable website layouts. But, to take our website layouts to the next level, we need to be able to break out of the normal element flow and customize it to our liking.

To achieve this, we will focus on 2 important concepts:

  1. The position property
  2. CSS Flexbox

By using position and Flexbox, we're able to easily create more complex CSS layouts with less CSS markup!

Position

The first one on the list is the position property. Even though it's just one CSS property, it can have a huge impact on how your elements are positioned on the page. So far, all of the HTML elements we've been working with have use default positioning: position: static. But what if we want to have overlapping elements? What about elements that stay on the screen? We can implement these by changing the position of our elements. Linked below is a video that does a great job of explaining the basics of the 5 different position property values available to us in CSS.

As a summary for the video, here are the 5 position values that we can use to position our elements:

  1. static: The default position value, meaning it follows the default HTML document flow. All of your HTML elements are positioned static until set to one of the other 4 values below.
  2. relative: No difference on its own, but unlocks the use of top, bottom, left, and right properties to position your element relative to where it would be if it were positioned static. Primarily used as a reference point so that other elements can position themselves relative to the element with position: relative.
  3. absolute: Removes the element from the document flow completely, meaning that other elements in the normal document flow will ignore this element, allowing for overlapping. It will position itself relative to its nearest non-static positioned parent, or the <body> element if none exist.
  4. fixed: Works exactly like absolute, except the element is positioned relative to the viewport (browser window). fixed positioned elements will always stay on the same spot on your browser, even if you scroll.
  5. sticky: Works like a combination of relative and fixed. sticky positioned elements will be positioned relative, but will stick to a position on the screen once you scroll past it.

Side Note: z-index

As you saw in the linked video about CSS Position, with some position values like position: absolute, sometimes multiple elements can overlap. By default, absolute positioned elements are placed over static positioned elements. But there is a way to override this default behavior: by using the z-index property. You can read about it in this page: The z-index Property

Flexbox

As you could tell from our previous section, the CSS position property is very useful for positioning individual elements. But, CSS layout tools like Flexbox are much better at creating flexible layouts involving multiple elements.

Bookshelf Analogy

In a nutshell, Flexbox is a way to organize elements on your webpage similar to how we organize books on a shelf. You place one book after another from left to right, and when you run out of space, you go to the next shelf below it. This is how Flexbox works, but there’s a lot of different things we can do to make our Flexbox layouts more interesting than a bookshelf is. Below is another link to a great video that helps explain Flexbox in a quick and simple manner.

Following our bookshelf analogy, Flexbox works by creating a flex container (bookshelf), and one or more flex items (books). We can configure the bookshelf to have vertical shelves or store books in reverse order. We can also tell our bookshelf how we want to space the books out when there’s extra space, and if we want to create a new shelf if we run out of space. We can even decide how we want to space out our shelves when we have multiple.

What about Grid?

Flexbox and Grid are more recent additions to CSS, and they help you layout elements in either a 1D or 2D way, respectively. Although Flexbox is extremely powerful and very useful for creating layouts in either rows or columns, you will eventually need to pick up Grid as well.

Unfortunately, we won't be going into detail on Grid in this class, there's just not enough time. Don't worry, though. Many of the concepts in Flexbox also apply to Grid, so you'll be able to pick it up quickly after gaining experience with Flexbox.
